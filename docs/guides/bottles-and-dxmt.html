How It Works

Bottles & DXMT, explained

What a Wine bottle is, how to make one, and why DXMT is the layer that makes your GPU speak Windows.

What's a bottle?

A bottle is a self-contained Windows environment — its own C:\ drive, registry, and settings — that your games install into and run from. MoltenVR creates bottles with its own downloaded, managed Wine build (WhiskyWine/GPTK), so you don't need Whisky, CrossOver, or any other Wine app. Already have one of those? MoltenVR can use your existing Whisky, CrossOver, or Wineskin bottles too.

What's DXMT?

Windows games draw graphics with Direct3D 11. Macs speak Metal. DXMT is the translation layer that converts D3D11 calls to Metal in real time, so the game renders natively on Apple Silicon. MoltenVR installs it into the bottle as native-flavour DLLs with a global override — games pick it up automatically, nothing to configure per game.

Create a bottle with DXMT — 4 clicks

  1. Open the Bottles tab, type a name, and click Create. MoltenVR fetches its managed Wine on first use and initialises the bottle.
  2. Select the bottle and click Install DXMT. Wait for the green installed check.
  3. Click Register OpenXR Runtime so VR titles find MoltenVR instead of looking for SteamVR.
  4. Click Install Steam, then Launch Steam and log in. (Launch .exe… runs any Windows game directly, no Steam needed.)

Done when the bottle row shows DXMT ✓  Runtime ✓  Steam ✓. Hit Rescan any time to refresh the checks.

DXMT or D3DMetal?

The Bottles tab offers both. They do the same job — D3D11 on Metal — via different routes:

LayerWhat it isWhen to use it
DXMT Open-source D3D11 → Metal translation (3Shain/dxmt). Installed as native DLLs with a global override. Default choice. Verified with Beat Saber, Half-Life: Alyx (via DXVK), and the rest of the tested list.
D3DMetal Apple's translation layer from the Game Porting Toolkit, shipped in the managed Wine build. Fallback if a specific game misrenders or crashes under DXMT. One click to switch — try it per bottle.

Good to know

One bottle is fine for most people. Steam and all your Steam VR games can live in a single bottle. Make a separate bottle when you want isolation — e.g. one for a heavily modded Beat Saber and one kept clean.

Performance is baked in. Bottles launch with esync enabled (WINEESYNC=1) for fast in-game and Steam-download performance — the app manages this for every launch, nothing to set.

Existing bottles keep their layout. Point MoltenVR at a Whisky/CrossOver/Wineskin bottle and it installs DXMT and the runtime into it without touching your games.

Troubleshooting. Steam won't start or downloads crawl after an update? Use Update Wine (fixes Steam) in the Bottles tab. A game renders wrong? Swap DXMT ↔ D3DMetal and relaunch. Checks not updating? Rescan. Anything else — grab the bottle log from the Bottles tab and open an issue or ask in Discord.
